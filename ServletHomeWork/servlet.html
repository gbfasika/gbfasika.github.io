1, Can post requests be bookmarked? What are the problems? An http post can be bookmarked but since a bookmark only consists of the URL all the form parameter will be lost. This will often mean the web server does not know what to do with the request since it was expecting some form parameter. If we submit via get request, all the form parameter go into URL so bookmark will contain all the information needed for the webserver to rebuild the page a second time. 2 What is the purpose of request dispatching? Request dispatcher is an interface whose implementation define an object which can dispatches the request to any resource on the server. The requestDispatcher interface provide the facility of dispatching the request to another resource it may be html, servlet, or jsp. This interface can also be used to include the content of another resource also. It is one of the ways of servlet collaboration. There are two methods defined in the requestDispatcher interface. a, forward(ServletRequest request, ServletResponse response) and b, include(ServletRequest request, ServletResponse response). 3,What is the difference between redirect and request dispatch? When a servlet does a redirect, it is like asking the client to call someone else instead. In this case, the client is the browser not the user. The browser makes the new call on the user’s behalf after the originally requested servlet say sorry call this guy instead. The user sees a new URL in the browser. When a servlet does a request dispatch, it’s like asking a co-worker to take over working with a client. The co-worker ends up responding to the client, but the client doesn’t care as long as someone responds. The user never knows someone else took over, because the URL in the browser bar doesn’t change. 4, what is an attribute? An attribute is an object that is used to share information in web application. Attribute allows servlet to share information among themselves. Attribute can be Set and Get from one of the following. A, request B, Session C, Application. 5, What is the deference between attributes and parameters? The key difference between attributes and parameters is that an attribute is a variable of any type that is declared directly in a class while a parameter is a variable defined by the function that receive a value when it is called. 6, What are dangers of using attributes? • Not thread safe • Nor session attributes • Only request attributes thread safe 7, What does it mean to say that http is stateless?
